# Contexte : Projet Stam Acoustic — Démonstration IA pour PMI

## Résumé

Nicolas MAUVILAIN (Directeur SC, consultant IA indépendant) accompagne Stam Acoustic, PMI d'une cinquantaine de personnes en Alsace (acoustique, plâtrerie, sols). Le projet est initié suite à une affaire perdue dont le retour client indiquait que leur description technique était *"trop imparfaite et pas assez précise"*. L'objectif est de démontrer la valeur de l'IA à travers une maquette concrète sur la prescription technique, puis de présenter une roadmap IA adaptée à leur niveau de maturité. Une réunion de restitution est à organiser avec Pierre-Alain Mendler (gérant de Stam Acoustic).

## 2. Audit des mémoires techniques (synthèse)

**Documents analysés (5)** : Mém. Technique + Mém. Environnemental MPE Wasselonne (Lot 11 Baffles), Note métho Golf Wantzenau (Lot 11 Plâtrerie/Plafonds), Note métho JEHU Usine PCO Socomec (Lot 12 Sols collés), Note métho Crédit Mutuel Equity (.doc, analysé par ChatGPT).

### Notation estimée (vue MOE/AMO)

| Critère | Note |
|---|---|
| Crédibilité technique | 8,5/10 |
| Respect du dossier | 9/10 |
| Moyens humains | 8/10 |
| Développement durable | 8/10 |
| **Différenciation** | **5/10** |
| **Qualité rédactionnelle** | **6/10** |
| **Mise en forme** | **4/10** |
| **Personnalisation** | **5/10** |
| **Note globale** | **≈ 70–75/100** |

> **Diagnostic** : "Vous avez la matière d'un acteur 8,5/10 sur la technique, mais vous la présentez comme un 6/10 sur la forme."

### 8 faiblesses majeures

1. **Auto-sabotage PAQ** — Wasselonne : *"Nous n'avons pas de plan d'assurance qualité formalisé, les procédures sont orales."* Critère noté 10/50 → probablement 0–1 point, potentiellement éliminatoire.
2. **Copier-coller massif** — blocs identiques mot pour mot sur 3 documents. Le MOE le voit.
3. **Aucune personnalisation au chantier** — méthodologie quasi identique partout alors que les enjeux sont totalement différents (restaurant ≠ usine ≠ tertiaire occupé ≠ école).
4. **Aucune alternative technique** — partout "nous respectons les prescriptions". Jamais une variante chiffrée.
5. **Formulations génériques** — "nous participerons aux réunions", "le conducteur vérifiera", "nous respecterons les DTU" → rapportent 0 point.
6. **Coquilles et incohérences** — "32 années" / "33 années" / "21 ans" dans le même doc ; JEHU déclaré membre de la FIPS (plafonds) alors qu'il pose des sols ; "Plafistes" (3 fois).
7. **Aucun chiffre concret** — pas de KPI qualité, pas de bilan carbone global, pas de planning Gantt, pas d'organigramme nominatif.
8. **Mise en forme datée** — gros pavés texte, monocouleur, aucun pictogramme, aucun schéma.

### Atout sous-exploité
4 sociétés (STAM, JEHU, Modul'Est, Delta Studio) — **6,5 M€ CA 2025** — multi-corps. Mentionné en passant, absent de certains docs. C'est un argument différenciant majeur.

### Plan d'action 10 points

**Quick wins (1–2 semaines, +5 à +10 pts) :**
1. Supprimer la phrase PAQ + remplacer par version constructive
2. Corriger erreurs factuelles (FIPS→SFEC, 33 ans partout, coquilles)
3. Bandeau couverture "Pourquoi nous choisir" standardisé
4. Relecture croisée systématique avant envoi

**Structurels (1–3 mois, +5 à +10 pts supplémentaires) :**
5. Formaliser un PAQ écrit (1–2 pages)
6. Refondre le template mémoire : 20% tronc commun / 40% spécifique chantier / 40% visuel
7. Bibliothèque visuelle (organigramme, planning type, schémas pose, photos)
8. Bibliothèque variantes techniques par ouvrage (acoustique/plâtrerie/sols)
9. Calcul bilan carbone chantier automatisé (DPGF × FDES INIES)
10. KPIs qualité internes chiffrés

### 6 cas d'usage IA pour la démo

| # | Cas d'usage | Effort | Impact note |
|---|---|---|---|
| 1 | Relecture qualité : coquilles, incohérences, copier-coller | Faible | +2 à +5 pts |
| 2 | Vérification conformité grille MOE : chiffres + référence + engagement | Faible | +5 à +10 pts |
| 3 | Prescription enrichie depuis DCE : variantes perf/coût/CO₂ + FDES | Élevé | +10 à +15 pts |
| 4 | Bilan carbone chantier auto : DPGF × FDES INIES | Moyen | +3 à +5 pts |

**La démo gagnante** : combiner **#1 + #2 + #3** sur Wasselonne (avant/après). Gain visible en points concrets sur la grille.

## 3. Stratégie de restitution et roadmap IA

### Risque identifié
Si la valeur se limite à "prompter ChatGPT", Pierre-Alain peut le faire seul → perte de pertinence immédiate. La différenciation ne peut pas reposer sur un prompt simple.

### Déroulé réunion (30–45 min)
1. **Restitution de l'audit** (20–30 min) — discussion, critique constructive
2. **Démonstration DIY** — montrer qu'ils auraient pu obtenir cet audit seuls en 5 minutes
3. **Ouverture sur la roadmap** — présentation des 3 niveaux de maturité

### Roadmap IA — 3 niveaux

**Niveau 1 — Usage individuel de base**
- Prompts ChatGPT/Claude : résumés, corrections, mise en forme
- Aucune infrastructure requise
- Point d'entrée accessible à tous

**Niveau 2 — Claude Projects (ou équivalent)**
- Espace dédié par client avec docs chargés (ex : chantier.xyz)
- Templates définis + bibliothèque de schémas
- Règle rédactionnelle : toujours note de synthèse courte + note détaillée longue

**Niveau 3 — Agents IA en entreprise**
- Automatisation : avant-vente → devis → factures achat → facturation client
- Workflow cible : email entrant → extraction nom client/projet → génération devis auto
- Approche multi-agents : agent générateur + agent critique (itère jusqu'à élimination des objections)
- Second cerveau entreprise : capitalisation accessible via requêtes IA (continuité si collaborateur absent)

### Gestion des coûts IA
- Consommation réelle Nicolas : ~2,50 €/semaine sur forfait 10 €/mois
- Règle : "saucissonner" l'information en mailles suffisamment petites
- Piste LLM local pour tâches courantes, LLM payant pour tâches complexes
- **Principe directeur** : ne pas construire un business sur un coût IA inconnu et non maîtrisé

### Contexte technique Stam — Batygest
- **Batygest = colonne vertébrale** du process (affaires, devis, factures) — mais ne génère pas de documents
- Documents stockés dans le cloud séparément
- Workflow = Batygest + cloud ; l'IA peut se connecter aux sorties cloud
- Intégration à étudier via API ou imports/exports

### Préparation réunion Pierre-Alain

**Slide lexique à préparer** : 5 concepts — LLM, Token, Prompt + ChatGPT, Claude, Gemini

**Métaphore à utiliser** : *"L'IA = le stagiaire compétent mais qui a besoin de consignes très directives"*

**Distinction note technique vs note de conviction** :
| Note technique | Note de conviction |
|---|---|
| Couverture fonctionnelle, conformité, références | Modernité, confiance, envie de travailler ensemble |
| Ce qu'on sait faire | Ce que ça donnera comme résultat pour eux |

La note de conviction doit montrer : suivi de commande étape par étape, dates prévisionnelles, plans d'action visibles.

**Veille à mettre en place** :
- Fournisseurs critiques : santé financière (Altares, Creditsafe)
- Évolutions techniques métier : nouvelles solutions acoustiques, DTU, matériaux
- Veille commerciale : appels d'offres, concurrents, prix marché

## 4. Template prompt IA — Mémoire technique (système prêt à l'emploi)

**À charger dans Claude Projects comme instruction permanente :**

```
Tu es l'assistant de rédaction de Stam Acoustic. Règles fondamentales :
- 20% tronc commun / 40% spécifique chantier / 40% visuel
- La conviction prime sur la conformité
- NE JAMAIS écrire : "Nous n'avons pas de PAQ formalisé" / "Nous respecterons les DTU"
- TOUJOURS inclure : PV acoustiques + variante technique chiffrée + engagement qualité mesurable + bilan CO₂ estimé

Structure imposée :
1. Couverture "Pourquoi nous choisir" (6 arguments)
2. Ce chantier vu par Stam (enjeux spécifiques)
3. Proposition technique : prescrit + variante(s) perf/coût/CO₂
4. Méthode : planning + organigramme nominatif + engagement qualité
5. Bilan carbone (quantités × FDES INIES)
6. Références similaires (2 minimum)
```

**5 commandes disponibles** :
1. Génération complète (CCTP en entrée → mémoire 4–6 pages)
2. Relecture qualité (mémoire existant → corrections + version améliorée)
3. Personnalisation rapide (type chantier + contrainte + durée + surface)
4. Variante technique (ouvrage prescrit → 2 variantes perf/coût/CO₂)
5. Bilan carbone (liste matériaux + quantités → bilan CO₂ + rédaction section env.)

**Coût estimé** : ~3 500 tokens par mémoire = 0,02–0,05 € par document via API. Gratuit en abonnement Pro (20 €/mois).

## 5. Test génération devis — ROI chiffré

**Cas test** : Chantier plafonds acoustiques 800 m² — 63 080 € HT (plafonds Ecophon + Knauf + ossature + isolation)

**Prompt prêt** : données client + quantitatif → devis professionnel structuré (en-tête, tableau, conditions générales, zone signature)

**ROI estimé** :
- Aujourd'hui : 30 devis/mois × 45 min = **22,5 h/mois**
- Avec IA : 30 devis × 5 min = **2,5 h/mois**
- **Gain : 20 h/mois libérées** ≈ valeur d'un mi-temps administratif

**Workflow cible Niveau 3** :
```
Email entrant (nom client + projet)
  → Récupération DPGF depuis Batygest (export CSV)
  → Génération devis par IA
  → Agent critique : vérification cohérence prix/quantités
  → Validation humaine 2–5 min
  → Signature et envoi
```

```
✓ 33 ans d'expérience (depuis 1993)
✓ 100 % équipes internes — aucune sous-traitance
✓ Collectif STAM : 4 sociétés / 6,5 M€ CA / interface multi-lots
✓ Déchetterie propre 10 flux + filière ReStart Tarkett + Valobat
✓ Qualifications Qualibat 6612 + 4132
✓ Implantation locale (Schiltigheim, EMS)
```

**Personnalisation par type de chantier** :
| Type | Enjeu spécifique |
|---|---|
| Restaurant haut de gamme | Ouverture saisonnière impérative, finitions visibles premium |
| Tertiaire occupé | Nuisances acoustiques, horaires décalés, maintien d'image |
| Industriel/usine | Coactivité, sécurité process, interfaces techniques |
| Équipement public | Phasage hors présence usagers, contraintes réglementaires |

**Reformulations clés** :
| Avant | Après |
|---|---|
| "Nous participerons aux réunions." | "Un point hebdomadaire interne avant chaque réunion de chantier — mesures correctives proposées en séance." |
| "Le conducteur vérifiera." | "Fiche levée de réserves hebdo, taux cible >90% en 1er passage." |
| "Nous respecterons les DTU." | (à supprimer) |
